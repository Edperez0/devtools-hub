Password Security Best Practices for Developers

By Eduardo Perez • January 25, 2025 • 9 min read

Password security is critical. One breach can compromise thousands of users. Here's what every developer must know about handling passwords correctly.

Never Store Plain Text Passwords

This should go without saying, but it still happens. Always hash passwords before storing them. No exceptions.

Use Proper Hashing Algorithms

Not all hashing is equal. Use these modern algorithms:

  • bcrypt - Industry standard, built-in salting
  • Argon2 - Winner of Password Hashing Competition
  • scrypt - Memory-hard, resists GPU attacks

Never use MD5 or SHA-1 for passwords. They're too fast and easily cracked.

Password Generation Best Practices

Strong passwords should have:

  • Minimum 12 characters (16+ recommended)
  • Mix of uppercase, lowercase, numbers, symbols
  • No dictionary words or common patterns
  • High entropy (randomness)

Implement Rate Limiting

Prevent brute force attacks by limiting login attempts. Lock accounts after 5-10 failed attempts and implement exponential backoff.

Use Salting

Add random data (salt) to passwords before hashing. This prevents rainbow table attacks and makes each hash unique even for identical passwords.

// Good - using bcrypt with automatic salting
const bcrypt = require('bcrypt');
const saltRounds = 12;
const hash = await bcrypt.hash(password, saltRounds);

Multi-Factor Authentication

Passwords alone aren't enough. Implement 2FA with TOTP (Time-based One-Time Passwords), SMS codes, or hardware keys.

Password Reset Security

  • Use time-limited, one-time tokens
  • Invalidate tokens after use
  • Don't reveal if email exists
  • Require email verification

Common Mistakes to Avoid

  1. Client-side only validation
  2. Weak password requirements
  3. Storing security questions (they're passwords)
  4. Sending passwords via email
  5. Not expiring password reset tokens
