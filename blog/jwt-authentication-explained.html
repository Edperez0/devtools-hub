JWT Authentication: A Complete Guide for Web Developers

By Eduardo Perez • October 15, 2024 • 9 min read

JWT (JSON Web Token) is the standard for secure authentication in modern web applications. Understanding how they work is essential for building secure APIs.

What is a JWT?

A JWT is a compact, URL-safe token containing three parts separated by dots:

header.payload.signature

Header

Contains token type (JWT) and signing algorithm (HS256, RS256, etc.)

Payload

Contains claims (user data, permissions, expiration time). Not encrypted - don't store sensitive data.

Signature

Verifies the token hasn't been tampered with. Created using header, payload, and secret key.

How JWT Authentication Works

  1. User logs in with credentials
  2. Server validates and generates JWT
  3. Client stores JWT (localStorage, cookie)
  4. Client sends JWT with each request (Authorization header)
  5. Server verifies signature and grants access

Security Best Practices

  • Use HTTPS always
  • Set short expiration times (15 minutes access, 7 days refresh)
  • Store tokens securely (httpOnly cookies preferred)
  • Implement token refresh mechanism
  • Validate signature on every request
  • Use strong secret keys (256+ bits)

Common Mistakes

  • Storing sensitive data in payload (it's Base64, not encrypted)
  • No expiration time (tokens valid forever)
  • Weak secret keys
  • Not validating tokens server-side
  • Storing in localStorage (XSS vulnerable)

JWT vs Session Auth

JWT pros: Stateless, scalable, works across domains

JWT cons: Can't revoke easily, larger than session IDs

Sessions pros: Easy to revoke, smaller

Sessions cons: Requires server-side storage, harder to scale
